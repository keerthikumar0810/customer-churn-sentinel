[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture diagram to see how data flows. Every query from our user begins at the START node and immediately enters the orange security_checkpoint node. 
(pause)
If any security violation or prompt injection is detected, the workflow routes to the security_handler to reject the request safely.
(pause)
If clean, the coordinator_agent takes over. It acts as our central brain, calling two specialized sub-agents via AgentTool: the risk_analysis_agent, which fetches live data from our database using our custom FastMCP server tools, and the retention_strategy_agent, which references our compliance rules to draft the optimal retention package.
(pause)
Once the analysis is complete, the decision_gate evaluates the risk score. If the score is high—specifically point eight or above—the workflow routes to the blue human_review_node to pause execution and request human sign-off. Otherwise, low-risk cases are auto-approved and sent directly to the final_output.

[3:00 — SECURITY & MCP]
This level of protection is possible because of our custom security checkpoint, which scrubs PII like emails, phones, SSNs, and credit cards, blocks prompt injections, and logs every event to a structured JSON audit log. On the data side, our custom FastMCP server connects the agents directly to our database, giving them read-only access to support tickets and retention policy boundaries in real-time.
